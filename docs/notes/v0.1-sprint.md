# aus-roads v0.1 sprint plan (committed 2026-06-01)

## User story

Install the `sa-map-v0.1.apk`. On first launch, a card says "Download the South
Australia map pack (~150 MB)". Tap **Download**. The pack lands in app-private
storage. The app opens to a vector map of South Australia centered on Adelaide
at zoom 11. Pinch-zoom to street level, two-finger pan across the state. The
map works in airplane mode from that point on.

Tap a road → bottom sheet with road name + suburb + (if mapped) speed limit.
Tap a POI → name + type. Search bar accepts "Adelaide Oval", "Mount Gambier",
"Ceduna 5690" → results list, tap → fly camera. Long-press the map → drop a
named pin, persists across restarts.

Bottom bar: **Map / Pins / Settings**. Settings shows the pack version, a
re-download button, an **About** entry with full attribution (OpenStreetMap,
Geofabrik, etc.), and a privacy statement.

What does NOT work in v0.1: no GPS dot, no my-location button, no traffic
overlay, no routing, no outback warnings, no community reports, no network
calls of any kind after the map-pack download.

## Acceptance criteria

- [ ] Signed debug + release APK builds from a clean clone via `./gradlew assembleRelease`.
- [ ] APK installs and launches on Android 10, 12, 14 (minSdk 26, targetSdk 35).
- [ ] Vector base map covering all of South Australia renders from a local MBTiles file.
- [ ] App is fully functional in airplane mode after the pack is installed.
- [ ] Pinch-zoom, pan, double-tap zoom work at 60 fps on a Pixel 6a.
- [ ] Tap on a road reveals its name; tap on a POI reveals its name and category.
- [ ] Free-text search returns ≥1 result for every SA suburb in the test set.
- [ ] Tapping a search result flies the camera.
- [ ] Long-press drops a named pin; pins survive app kill and `pm clear`.
- [ ] NO `ACCESS_*_LOCATION` permission is requested.
- [ ] NO `INTERNET` permission in the shipped v0.1 manifest.
- [ ] Settings shows pack version, size, install date, re-download control.
- [ ] About screen lists every third-party dataset and license.
- [ ] No analytics SDK, no crash reporter, no Firebase, no Google Play Services.
- [ ] Cold start to interactive map ≤ 3 s on Pixel 6a.
- [ ] No crashes across a 30-minute manual smoke session.

## Tracks

- **Track A** — Project foundation (Gradle, version catalog, build-logic, CI)
- **Track B** — Map rendering (MapLibre, gestures, attribution, tap-identify)
- **Track C** — Map pack pipeline (PBF → MBTiles, manifest, in-app downloader)
- **Track D** — Search & POI (SQLite FTS5)
- **Track E** — UI shell (bottom nav, pins screen, settings screen)
- **Track F** — Permissions & privacy (no-INTERNET, no-location, CI gate)
- **Track G** — Test & QA (unit, instrumented, manual checklist, perf pass)

## Sequence

1. **A** first (Gradle skeleton, version catalog, build-logic, CI).
2. **F1 / F2** (no-INTERNET manifest, privacy stub) in parallel with A.
3. **E1** (Compose bottom-nav scaffold) right after A2.
4. **C1** (PBF → MBTiles pipeline) produces a real SA pack; then **B2**
   (MapLibre renders the local pack) and **D2** (search index).
5. **B3 / B4 / B5** (gestures, tap-identify, attribution) in parallel.
6. **C4** (in-app downloader) and **D3** (search UI) merge last; they wire
   the user-facing seams.
7. **G1 / G2** (tests) written alongside their target modules, not at the end.
8. **G3 / G4** (manual QA + perf) are the final gate before ship.

## Sprint-1 3-day cut (morale milestone)

- **Day 1**: A1-A2, B1, E1. App opens to a blank `MapView` over a hard-coded
  Adelaide-metro MBTiles in `assets/`. Pan/zoom work. Nothing else.
- **Day 2**: B3, B5, F1, F2. App is provably offline-only. No search, no pins.
- **Day 3**: C1 spits out a real SA-wide MBTiles, B2 swaps the asset in, E3
  is a stub Settings screen. The user can fly across SA.

By end of Day 3, benbi has a real, **impressive** APK: "an offline vector map
of the entire state of South Australia on my phone, no internet, no location,
no tracking." Everything else is polish on top of that proof.

## Ship gate

1. Every box in the acceptance criteria is checked.
2. A signed release APK installs on a stock Pixel 6a and demonstrates the
   full user story in airplane mode.
3. CI is green on `main`; no `INTERNET` / `ACCESS_*_LOCATION` in the merged
   manifest.
4. `docs/notes/qa-v0.1.md` and `docs/notes/perf-v0.1.md` exist and are
   signed off.
5. A 60-second screen recording of the full v0.1 flow is attached to the
   release tag.

## v0.1 explicitly out of scope

- Traffic overlay (v0.2+)
- GPS / location permission
- Routing, turn-by-turn, ETA
- Outback road warnings
- Community hazard reports
- Police / emergency presence
- Live congestion / commercial traffic API
- Network calls of any kind after the one-time map-pack download
- iOS, web, Wear OS, Auto, tablet-optimized layout
