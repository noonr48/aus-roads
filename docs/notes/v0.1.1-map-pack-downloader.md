> **STATUS: SPEC / NOT YET IMPLEMENTED.** This document describes a planned design for the in-app map pack downloader. As of v0.1.1, the map pack is a bundled asset loaded from `assets/maptile/`. None of the infrastructure described below (MapPackManager, WorkManager, Ktor, INTERNET permission) has been built.

# v0.1.1 — In-app map pack downloader

Sub-agent architecture handoff. Replaces the v0.1 placeholder `MapScreen`
and `SettingsScreen` with a real offline map pack downloader that respects
the v0.1 privacy posture (with one controlled, audited exception for
`INTERNET`).

References: `docs/adr/0003-map-pack-format.md`, `docs/notes/privacy.md`,
`docs/notes/v0.1-sprint.md` (Track C4, Track F3).

## 1. Privacy posture change and decision (the one hard question)

We add `INTERNET` to v0.1.1. Concretely: option (a) — declare
`<uses-permission android:name="android.permission.INTERNET" />` in the
`AndroidManifest.xml` of `:app` at `android/app/src/main/AndroidManifest.xml`.

Option (b) — hand the URL to `DownloadManager` or a system browser via
`Intent.ACTION_VIEW` — is rejected because (i) we cannot verify the SHA-256
inside our process (the file lands outside the app sandbox and the manifest
hash is the only thing that defends against a CDN compromise), (ii) a
200–400 MB zip is unreliable to resume in a system browser, and (iii) the
user would lose the in-app progress UI. The browser/DownloadManager path
does not actually preserve the v0.1 promise either: those components
still hit the network; we have just moved the permission to a different
UID.

### What the CI privacy audit must allow in v0.1.1

The v0.1 gate (Track F3) is `aapt dump permissions | grep -E
'INTERNET|ACCESS_(FINE|COARSE)_LOCATION'` and fails on a match (see
`docs/notes/privacy.md` §v0.1 and `docs/notes/v0.1-sprint.md` §Ship gate).
In v0.1.1 we replace it with a structured allowlist in
`tools/privacy-audit/`:

1. **Permission + variant allowlist.** `INTERNET` is permitted only on a
   new `withNetwork` build flavor. The `offline` flavor keeps v0.1
   behavior for sideload distribution. `ACCESS_FINE_LOCATION` and
   `ACCESS_COARSE_LOCATION` remain hard-fail in every variant.
2. **Endpoint allowlist.** The only `HttpClient(OkHttp)` instance in the
   app is constructed by `MapPackManager` (see §11). The audit greps
   `:app` and all `:feature:*` modules for `HttpClient(` and fails the
   build on a second instance. The Ktor engine has a request-time host
   check that rejects anything outside the R2 primary or GitHub Releases
   mirror named in ADR 0003.
3. **Trigger allowlist.** `MapPackManager.startDownload()` is the only
   code path that calls `client.get(...)`. It is gated on a user tap
   (Settings → Download, the first-launch card, or Re-download) and is a
   no-op while a fresh pack is installed.
4. **CI hook.** `tools/privacy-audit/run.sh` runs from `./gradlew check`
   and from CI on every PR.

`docs/notes/privacy.md` §v0.1 is updated to: "v0.1.1 ships a one-time,
user-initiated, foreground map-pack download. The permission is
`INTERNET`; the audit allowlist in `tools/privacy-audit/` is the only
thing that can grant it, and it can only be granted to `MapPackManager`."

## 2. Download library — confirmed

Ktor 2.3.13 client with the OkHttp engine, as already pinned in
`android/gradle/libs.versions.toml` lines 39 and 97–101. Range support
works through the `Range` request header on
`client.get(...) { headers { append(HttpHeaders.Range, "bytes=$n-") } }`
(the `headers` builder on `HttpRequestBuilder` is the documented surface
in `https://ktor.io/docs/client-requests.html` §Set request parameters).
OkHttp under the hood natively supports range requests and `206 Partial
Content` with `Content-Range` (see
`https://github.com/square/okhttp/issues/1551`).

Plugins to install on the `HttpClient`:

- `ContentNegotiation` + `Json` (kotlinx.serialization). Needed only for
  the manifest. The zip body is consumed as a `ByteReadChannel`, not
  deserialized.
- `Logging` (SLF4J → Android `Log` via `Logger.DEFAULT` in debug builds
  only). Stripped in release by R8.
- `HttpRequestRetry` with `retryOnServerErrors(maxRetries = 3)` and an
  exponential `delay` (1s → 2s → 4s, capped at 30s) for `5xx` and
  `IOException`. Does not retry on `404` or `416 Range Not
  Satisfiable`.

## 3. HTTP semantics

### Manifest fetch (cheap refresh)

```
GET https://cdn.aus-roads.example/aus-roads/sa/latest.json
If-None-Match: "<etag from last successful fetch>"
If-Modified-Since: "<http-date from last successful fetch>"
Accept: application/json
User-Agent: aus-roads/<versionCode> (<versionName>; Android <sdkInt>)
```

Response: `200` with body (deserialize into `PackManifest`) or
`304 Not Modified` (reuse the cached manifest, refresh ETag/Date).
Cache the ETag and Last-Modified in `manifest-cache.json` next to the
install dir.

### Pack zip fetch (resumable)

We download a single `sa-pack-<packVersion>.zip` containing
`manifest.json`, `tiles.mbtiles`, `routing/valhalla_tiles.tar`, and
`search.db`. The zip is a transport container; the manifest inside is
the authoritative source for component hashes.

```
GET https://cdn.aus-roads.example/aus-roads/sa/sa-pack-2026-06-01.zip
Range: bytes=<n>-          // n = current file size on disk, 0 on first attempt
User-Agent: aus-roads/<versionCode>
```

Responses:

- `200 OK` (server ignored Range — happens on some CDNs) — discard
  partial, restart from 0.
- `206 Partial Content` with `Content-Range: bytes <n>-<total-1>/<total>` —
  append to the partial file.
- `416 Range Not Satisfiable` — partial is longer than the new pack
  (unlikely); delete partial, restart from 0.
- `404` / `410` — the pack version no longer exists; surface a hard
  error suggesting the user retry the manifest.
- `5xx`, `IOException`, `SocketTimeoutException` — back off and retry
  per `HttpRequestRetry`.

The partial file lives at `mappacks/au-sa/.partial/sa-pack-<version>.zip`
(dot-prefix hides it from the eviction walker). On successful verify +
extract, the partial is deleted.

Eviction transitions: on entering `Installed`, delete the directory that
used to be `current` (if any) and write the new `current.json` /
`previous.json`. On process start, walk the install dir and delete any
`v*` directory not referenced by `current.json` or `previous.json`.

## 5. Concurrency — where the download runs

**WorkManager `OneTimeWorkRequest` with `setExpedited()` and a
long-running `CoroutineWorker` that promotes itself to a foreground
service of type `dataSync`.** Rationale:

- The user is told that the download is happening (a persistent
  notification "Downloading SA map pack — 47%" + a live progress bar in
  the Settings screen). That is the canonical user-initiated download
  pattern Google expects; see the Android 14 FGS type rules at
  `https://developer.android.com/about/versions/14/changes/fgs-types-required`,
  which lists `dataSync` as the intended type for "data upload or
  download ... transfer data between a device and the cloud over a
  network".
- Play Store policy treats expedited WorkManager as a foreground
  service; the audit needs `FOREGROUND_SERVICE_DATA_SYNC` declared in
  the manifest (normal permission, auto-granted).
- The work runs on `Dispatchers.Default` inside `doWork()`, with the
  actual network I/O on `Dispatchers.IO` for the streaming write.
  Progress is reported via `setProgress(workDataOf("pct" to n))` and
  observed by `MapPackViewModel` via
  `WorkManager.getWorkInfosForTagLiveData(uniqueWorkName)`.
- The user can leave Settings; the worker keeps running with a
  notification. They can also cancel: a `Cancel download` button in
  Settings calls `WorkManager.cancelUniqueWork(uniqueWorkName)`.

Work request shape:

```kotlin
val req = OneTimeWorkRequestBuilder<MapPackDownloadWorker>()
    .setConstraints(
        Constraints.Builder()
            .setRequiredNetworkType(NetworkType.UNMETERED) // ADR 0003 default; cellular opt-in is v0.4
            .setRequiresStorageNotLow(true)
            .build(),
    )
    .setExpedited(OutOfQuotaPolicy.RUN_AS_NON_EXPEDITED_WORK_REQUEST)
    .setInputData(workDataOf("packVersion" to target.packVersion))
    .addTag(uniqueWorkName)
    .build()
WorkManager.getInstance(ctx).enqueueUniqueWork(
    uniqueWorkName,
    ExistingWorkPolicy.KEEP, // a second tap is a no-op while one is in flight
    req,
)
```

`uniqueWorkName` is `"map-pack-download-au-sa"`.

## 6. Storage layout on device — pointer file, not symlink

```
context.filesDir/
  mappacks/
    au-sa/
      current.json          // { "version": "2026-06-01", "installedAt": "..." }
      previous.json         // { "version": "2026-05-25", "installedAt": "..." } | null
      manifest-cache.json   // { "etag": "...", "lastModified": "..." }
      v2026-06-01/          // install dir, named per packVersion
        manifest.json
        tiles.mbtiles
        routing/valhalla_tiles.tar
        search.db
      v2026-05-25/          // rollback copy
        ...
      .partial/             // hidden from the eviction walker
        sa-pack-2026-06-08.zip
```

**Pointer file, not symlink.** A `current -> v2026-06-01` symlink would
work in `context.filesDir` (a real Linux filesystem, not scoped
storage) but adds two failure modes: (i) `AtomicFile` /
`File.renameTo` does not preserve symlink targets and we would need a
manual `Os.symlink` (API 21+, fine on minSdk 26) on every swap, (ii)
any code that walks the parent dir with `Files.walk` and follows
symlinks will double-count the pack size. A 60-byte JSON file read at
startup is simpler, faster, and survives anything the OS throws at it.
The trade-off is one extra file read on app start, which is
negligible.

## 7. Eviction policy

- **On successful new install:** the current install dir (if any)
  becomes the new `previous.json` value; the new dir becomes
  `current.json`; any other `v*` directory is deleted.
- **On every app start:** if `current.json` references a non-existent
  dir, revert to `previous.json` (or `null`); delete any `v*` dir that
  is not referenced by either. This is the partial-install defense — a
  process killed mid-`Extracting` leaves an orphan that gets cleaned up
  at next launch.
- **Storage budget:** at most 1 prior pack for rollback. We do not keep
  2 priors. We do not keep a "downloaded but never installed" pack
  (the partial zip is deleted after a successful extract).

## 8. Verification

For each component in `PackManifest.components`:

1. Open a `FileInputStream` for the extracted file.
2. Wrap in a `DigestInputStream(MessageDigest.getInstance("SHA-256"))`.
3. Read in 8 KB chunks, digesting per chunk.
4. Hex-encode the digest (lowercase, no `sha256:` prefix to match the
   manifest format) and compare to `component.sha256`.

On mismatch: delete the partial pack dir, the partial zip, and the
in-flight `current.json` (if speculatively written), then surface
`ChecksumMismatch` to the UI with a `Re-download` affordance. The
download does not auto-retry (no thundering herd on a broken CDN); the
user has to tap.

`signatures` is parsed but not verified in v0.1.1. When we sign
manifests (Ed25519, v0.4+ per ADR 0003) the verifier is a 30-line
addition in `MapPackManager.verifyManifest(manifest, publicKey)` and the
public key is bundled in `:offline:pack-api`.

## 9. Error states and their UI copy

| State | Surface |
|---|---|
| No internet | "Connect to Wi-Fi to download the SA map pack." (auto-retries on connectivity.) |
| Manifest unreachable / 5xx | "Couldn't reach the aus-roads CDN. Tap to retry." |
| `404` on `latest.json` | "No SA pack available yet. Check back after the next weekly build." |
| 3 retries exhausted on the zip | "Download interrupted. Tap to resume." |
| `minAppVersion` newer than us | "Pack requires aus-roads v0.2 or later. Update the app." |
| `minAndroidSdk` higher than device | "This device is too old for this pack (needs Android N)." |
| Disk full | "Need ~350 MB free, you have ~80 MB. Free some space and try again." |
| SHA-256 mismatch | "Pack corrupted. Re-downloading automatically." (then a manual Re-download button.) |
| `416 Range Not Satisfiable` | (silent — treated as restart from 0) |

All copy goes in `strings.xml`.

## 10. Module layout

We add two new modules:

- `:offline:pack-downloader` — Android library. Owns `MapPackManager`,
  the `MapPackDownloadWorker`, the file-system state, verification, and
  eviction. Depends on `:offline:pack-api`, `:core:common`, Ktor +
  OkHttp, WorkManager, kotlinx-serialization, kotlinx-coroutines. Has a
  Hilt module exposing `MapPackManager` as `@Singleton`. Kept separate
  from `:offline:pack-api` because that module is pure Kotlin and the
  downloader drags in Ktor / OkHttp / WorkManager.
- `:feature:settings` — Android library with Compose. Owns
  `MapPackViewModel` and the reworked `SettingsScreen`. Depends on
  `:offline:pack-downloader`, `:ui:designsystem`, hilt-navigation-
  compose.

`:app` depends on both and wires the navigation.

## 11. API surface — `MapPackManager`

Lives at
`android/offline/pack-downloader/src/main/kotlin/au/com/ausroads/offline/download/MapPackManager.kt`.

```kotlin
@Singleton
class MapPackManager @Inject constructor(
    private val ctx: Context,
    private val client: HttpClient,                 // Ktor + OkHttp, ContentNegotiation/Json
    private val dispatchers: AppDispatchers,
    private val workManager: WorkManager,
    private val clock: Clock,                       // injectable for tests
) {
    val installed: StateFlow<InstalledPack?>
    val inFlight: StateFlow<DownloadProgress?>

    suspend fun fetchLatestManifest(): ManifestFetchResult
    fun startDownload(force: Boolean = false)
    fun cancelDownload()
    fun currentPackDir(): File?
    suspend fun reconcileOnLaunch()
}

sealed interface ManifestFetchResult {
    data class Fresh(val manifest: PackManifest) : ManifestFetchResult
    data object Unchanged : ManifestFetchResult
    data class Failed(val reason: FetchFailure) : ManifestFetchResult
}

enum class FetchFailure { Unreachable, NotFound, ServerError, SchemaTooNew }

data class InstalledPack(
    val version: String,        // e.g. "2026-06-01"
    val sizeBytes: Long,
    val installedAt: Instant,
    val manifest: PackManifest, // re-read from disk for the Settings UI
)

data class DownloadProgress(
    val phase: Phase,
    val bytesDownloaded: Long,
    val totalBytes: Long?,
) {
    enum class Phase { FetchingManifest, Downloading, Verifying, Extracting }
}
```

The `HttpClient` is provided by a Hilt module and is the only
`HttpClient(...)` in the app — this is the constraint the privacy audit
greps for.

## 12. UI integration — `MapPackViewModel` and the new Settings screen

Lives at
`android/feature/settings/src/main/kotlin/au/com/ausroads/feature/settings/MapPackViewModel.kt`.

```kotlin
@HiltViewModel
class MapPackViewModel @Inject constructor(
    private val manager: MapPackManager,
) : ViewModel() {

    val state: StateFlow<MapPackUiState> = combine(
        manager.installed,
        manager.inFlight,
    ) { installed, inFlight ->
        MapPackUiState(
            installed = installed?.let {
                InstalledSummary(it.version, it.sizeBytes / 1_048_576, it.installedAt)
            },
            progress = inFlight?.let { renderProgress(it) },
        )
    }.stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), MapPackUiState())

    fun onDownloadClick() = manager.startDownload(force = false)
    fun onRedownloadClick() = manager.startDownload(force = true)
    fun onCancelClick() = manager.cancelDownload()
}

data class MapPackUiState(
    val installed: InstalledSummary? = null,
    val progress: Progress? = null,
) {
    val isWorking get() = progress != null
}

data class InstalledSummary(val version: String, val sizeMb: Long, val installedAt: Instant)
sealed interface Progress { data class Pct(val pct: Int, val label: String) : Progress }
```

## 13. Risks and mitigations

1. **CDN serves a corrupted pack and SHA-256 catches it.** `ChecksumMismatch`
   does not auto-retry; the user gets a Re-download button. A counter in
   `InstalledPack` makes a third corruption in 24 h surface "CDN appears
   unstable" and stop hammering. *(Low / High.)*
2. **Process killed mid-extract leaves the install dir half-populated.**
   `reconcileOnLaunch()` runs at app start, walks the install dir,
   deletes any `v*` not referenced by `current.json` / `previous.json`,
   and reverts to `previous.json` if `current.json` is stale. Map screen
   falls back to the placeholder card. *(Medium / Medium.)*
3. **FGS notification leaks coarse location via the OS surface.**
   Notification copy is "Downloading aus-roads SA map pack" with no
   coordinates; progress is bytes, not %. *(Low / Low.)*
4. **WorkManager expedited quota exhausted on a misbehaving device.**
   `.setExpedited(OutOfQuotaPolicy.RUN_AS_NON_EXPEDITED_WORK_REQUEST)` —
   download still runs, just without the FGS-priority lane. *(Low / Low.)*
5. **Privacy audit regression — a future PR adds a second `HttpClient`
   for telemetry or crash reporting.** Audit script fails the build on
   any `HttpClient(` outside `MapPackManager`; review checklist item
   "if you need network, talk to benbi first." *(Low / Critical — this
   is the one that protects the v0.1 promise.)*
